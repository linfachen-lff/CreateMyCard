# 能力范围

## 支持的卡片任务

此 skill 用于根据一句话生成紧凑 HarmonyOS GenUI 桌面卡片，尤其适合：

- 天气 + 日程速览：当前状况、下个会议、快捷动作。
- 家庭关怀/天气告警：温度、风险标签、呼叫或关怀动作。
- 会议/日程提醒：下个事件、时间段、房间/地点、专注动作。
- 睡眠/健康进度：状态、环形进度、时长、提醒动作。
- 倒计时/训练计划：事件倒计时、今日计划、打开计划动作。
- 设备/产品状态：产品图片/资源、电量、模式、播放、连接状态。
- 通用状态卡：一个主指标、一个辅助上下文、一个动作。
- 紧凑信息摘要：把一个实体/概念/地点/事件总结成卡片，而不是文章。

默认输出是使用 `ohos.a2ui.extended.catalog` 的单个 Form 服务卡片 surface。

## 支持的尺寸

### `2x2` / `160 x 160vp`

优先用于内容和组件能在 `160 x 160vp` 内完整放下的速览卡片。选择前先估算信息字段、受保护文本长度、主区域数量和可点击控件尺寸；如果核心信息能在 `2x2` 内无滚动、无截断、无拥挤地展示，就不要升级到 `2x4`。

- 通常承载 1 个主答案/主指标、1 个简短上下文和最多 1 个主动作。
- 适合字段少、文本短、组件数量少的状态、提醒、进度或动作卡。
- 即使是日程、设备、进度、关怀、媒体或产品状态等场景，只要内容放得下，也应保持 `2x2`。
- 睡眠状态、会议提醒、关怀/联系、倒计时 + 单条计划、耳机/低电/连接状态、防沉迷时长、天气 + 一个事件等单主题卡片，默认判为 `2x2`。
- 图片、图标、产品主体或一个 CTA 不构成升级理由；它们可以作为 `2x2` 的单个视觉锚点。

### `2x4` / `160vp（高）x 320vp（宽）`

仅当核心内容或组件在 `2x2` 中放不下，但能在 `160vp（高）x 320vp（宽）` 内保持紧凑卡片形态时使用。不要因为场景名称天然选择 `2x4`；必须由内容容量、受保护文本完整显示和组件热区预算驱动尺寸选择。

- 必须能指出 `2x2` 的具体失败点：受保护文本放不下、按钮热区不足、需要 2 到 3 个独立 pane 并列理解，或关键媒体无法与必要详情同时可见。
- 需要同时完整展示标题、时间/状态、2 到 3 个辅助字段和 1 个主动作，且字段筛选后仍不能安全放入 `2x2`。
- 需要媒体/主视觉与状态、详情、动作并列，且媒体承载关键信息；如果媒体只是装饰或单个视觉锚点，保持 `2x2`。
- 需要横向拆成 2 到 3 个 pane 才能保护关键文本和按钮热区；单一主题 + 一个上下文组不要升级。

## 不支持或有风险的请求

当用户要求以下内容时，说明限制并提供一个更小的受支持卡片替代方案：

- 完整页面、多屏流程、重型仪表盘或文章/详情页。
- 需要大量输入、复杂交互或大型动态列表的复杂表单。
- Form 子集 10 个支持组件之外的组件。
- 需要不受支持的绝对定位或 CSS 类属性的布局。
- 未在 `reference/data-capability/` 中声明的实时数据获取、设备 API，或应用未提供的宿主函数。
- 用户没有提供、且不能用本地/资源路径表示的图片或媒体。
- 无法在 `2x2` 或 `2x4` 中压缩而不损失用户核心需求的内容。

## 默认假设

- 卡片尺寸：默认选择 `2x2`，除非请求明显需要 `2x4`，并能说明 `2x2` 的具体失败点。
- 宽度预算：`2x2` 为 `160vp`，横版 `2x4` 为 `320vp`。
- 高度预算：`2x2` 和 `2x4` 都是 `160vp`。
- 协议：包含 `createSurface`、`updateComponents`、`updateDataModel` 的 JSONL。
- Catalog：`ohos.a2ui.extended.catalog`。
- 内容密度：`2x2` 有 2 到 3 个主区域；`2x4` 有 3 到 4 个主区域。
- 字段筛选：如果用户需求涉及太多数据字段，模型自主选择最重要的最多 4 项展示；优先保留用户明确点名的字段、时间/状态/主指标/动作所需字段，剩余字段用概括或省略处理，不要因此扩展成页面、长列表或超过 `2x4` 的密度。
- 交互：默认最多一个主动作，除非用户明确需要更多。
- 已在 `reference/data-capability/` 中声明的端侧数据能力属于支持范围；未声明能力不要编造。

## 替代策略

如果请求过宽，生成一个回答首要速览需求的聚焦卡片：

- 旅行计划卡 -> 下次旅行倒计时 + 天气 + 打开行程。
- 财务仪表盘卡 -> 今日支出/预算状态 + 风险标签 + 查看详情动作。
- 项目管理卡 -> 下个里程碑 + 进度 + 打开任务动作。
- 文章摘要卡 -> 标题 + 2 个属性 + 一行摘要 + 打开详情动作。
